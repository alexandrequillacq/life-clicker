# Principe 07 · Pièges à éviter

**En une phrase :** la plupart des idle échouent toujours pour les mêmes raisons. Les connaître = les éviter par design.

## Les pièges (et l'antidote)
1. **Grind sans récompense / murs de progrès** *(cause n°1 de churn)*. Le progrès ralentit sans échappatoire (pas de prestige, pas de nouveau système). → Antidote : chaque palier prend ~le même temps *relatif* ; un prestige ou une nouvelle phase à chaque ralentissement (cf. [04](04-prestige-reset.md), [02](02-nouveau-verbe-par-phase.md)).
2. **Que des plus gros chiffres, aucune nouvelle mécanique** (« tableur déguisé »). → Antidote : [02 · un nouveau verbe par phase](02-nouveau-verbe-par-phase.md).
3. **Pacing mal calibré** : trop vite = contenu épuisé ; trop lent = ennui. ⚠️ Attention : « trop lent » est *souvent* en fait un problème d'UI/tuto, pas de maths.
4. **Pay-to-win / monétisation agressive** (AdVenture Capitalist est le cas le plus cité). Contre-modèle loué : Leaf Blower Revolution (monnaie premium 100 % gagnable, rien ne gate le progrès). → Pour nous : pas de monétisation qui bloque le progrès. Idéalement pas de monétisation du tout (projet perso).
5. **Hors-ligne cassé/radin** (Cell to Singularity mesuré à ~0,46 % du taux en ligne → révolte des joueurs).
6. **Prestige punitif** (mauvaise courbe, ou reset forcé/trop fréquent → « tourner en rond »).
7. **UI qui noie / info-dump à l'onboarding.** Balancer tous les systèmes dans les 10 premières minutes → churn. *Un seul tooltip clarifiant le prestige a donné +19 % de rétention J7 dans un cas.* → Antidote direct : [01 · révélation progressive](01-revelation-progressive.md).
8. **Le mur de contenu** (long terme). Antidote : une *cadence* d'updates/événements.

## Comment on l'applique à IdleGame
- Notre révélation progressive règle naturellement les pièges #2 et #7.
- Décider tôt une **durée totale cible** pour calibrer le nombre de phases/prestiges (évite #1 et #3) — voir [08](08-rythme-pacing.md).
- **Pas de pay-to-win.**
